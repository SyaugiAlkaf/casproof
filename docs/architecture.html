Casproof · architecture

The unskippable on-chain action firewall for AI agents

Verify-before-act, enforced in the Casper VM: the verify decision and the value-bearing action run in ONE atomic call, so an off-chain agent cannot skip the check. The proof of computation is pluggable; the unskippable settlement gate is the contribution.
Centrepiece · in-VM enforcement gate
Verify + act in one atomic Casper VM call
Verify-THEN-act off-chain leaves a gap an agent can bypass; Casper's own attestation toolkit attests but has no enforcement primitive. Casproof closes the gap: the consumer cross-calls registry.require_quorum inside the same transaction that moves value — the action only exists if the check passed.

Honest about today's threat model
The signer set is owner-curated and attestations are not yet bound to proof-of-computation — a curated signer could in principle copy a published hash. Copy-resistance comes from binding attestations to TEE / zk receipts (roadmap policies above); slashing gives the curated set skin in the game today. The novelty is the in-VM enforcement gate, not the inference proof.
